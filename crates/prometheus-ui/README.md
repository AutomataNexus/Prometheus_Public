# prometheus-ui

Leptos 0.7 WASM client-side rendered frontend for Prometheus, styled with the
NexusEdge design system. Compiles to WebAssembly via Trunk and runs entirely in
the browser -- no server-side rendering. The server crate
(`prometheus-server`) serves the built artifacts and proxies API calls.

## Architecture

| Layer | Technology |
|-------|-----------|
| Framework | Leptos 0.7 (CSR mode) |
| Router | `leptos_router` 0.7 with `ParentRoute` auth guard |
| HTTP | `gloo-net` with bearer-token auth helpers (`api.rs`) |
| Build | Trunk (`trunk build --release`) targeting `wasm32-unknown-unknown` |
| Styling | Inline global CSS generated by `theme.rs` (no external CSS toolchain) |

State is managed with `RwSignal` throughout. Authentication tokens are stored
in `localStorage` under the key `prometheus_token` and attached to every API
request via the `api::auth_get` / `auth_post` / `auth_put` / `auth_delete`
helpers. Unauthenticated users are redirected to `/login` by the
`AuthenticatedLayout` guard.

## NexusEdge Design System

All styling is defined in `theme.rs` as a single `global_styles()` function
that returns raw CSS injected into the DOM via a `<style>` tag. No build-time
CSS toolchain is required.

**Typography.** Inter (300-700) for body text, JetBrains Mono for code.

**Color palette:**

| Token | Value | Usage |
|-------|-------|-------|
| Primary | `#14b8a6` (teal) | Buttons, links, active states |
| BG Cream | `#FFFDF7` | Page and card backgrounds |
| BG Warm Beige | `#F5EDE8` | Secondary backgrounds |
| Border Tan | `#E8D4C4` | Card and input borders |
| Terracotta | `#C4A484` | Metric accent values |
| Text Primary | `#111827` | Body text |
| Text Secondary | `#6b7280` | Muted / label text |

Six domain-specific accent colors (medical, finance, NLP, vision, industrial,
audio) are available via `theme::domain_color()`.

Status colors follow standard conventions: green (success), orange (warning),
red (error), blue (info).

**Layout.** Card-based with a fixed left sidebar (220px expanded, 64px
collapsed), sticky 64px header, and responsive breakpoint at 768px that hides
the sidebar and switches grids to single-column.

## Building

Prerequisites: Rust nightly toolchain with `wasm32-unknown-unknown` target,
and [Trunk](https://trunkrs.dev).

```sh
# Install Trunk if needed
cargo install trunk

# Development (with hot-reload)
cd crates/prometheus-ui
trunk serve

# Production build
trunk build --release
```

The output lands in `crates/prometheus-ui/dist/`. The server crate serves
these files as the SPA fallback.

## Leptos 0.7 Patterns

This crate targets Leptos 0.7, which differs from 0.6 in several ways that
affect every file:

- Import via `use leptos::prelude::*;` (not `use leptos::*;`).
- Control flow components: `use leptos::control_flow::{Show, For};`
- `Children` is `FnOnce` -- it cannot be called inside `<Show>`. Use CSS
  `display:none` to conditionally hide content that contains children.
- Type erasure uses `.into_any()` (there is no `leptos::View` type).
- `spawn_local` lives at `leptos::task::spawn_local`.
- The `>` operator inside the `view!` macro is ambiguous with HTML angle
  brackets; wrap comparisons in braces: `{ x > 1 }`.
